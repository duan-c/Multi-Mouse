# Multi-Mouse Godot Add-on

`addons/multi_mouse/` is the Godot-facing layer. It ships a single script,
`multi_mouse_manager.gd`, which exposes a `MultiMouse` node (`class_name
MultiMouse`). No autoloads are registered anymore—instance the node wherever you
need multi-cursor input and opt-in explicitly.

## Usage

1. Enable the plug-in via **Project → Project Settings → Plugins → Multi-Mouse**.
2. Add a `MultiMouse` node to your scene (or `MultiMouse.new()` at runtime).
3. In `_ready()` call:
   ```gdscript
   var hwnd = 0 # or DisplayServer.window_get_native_handle(...)
   $MultiMouse.attach_to_window(hwnd)
   $MultiMouse.enable()
   $MultiMouse.motion.connect(_on_motion)
   $MultiMouse.button.connect(_on_button)
   $MultiMouse.device_connected.connect(_on_device_connected)
   ```

The node simply wraps the native `MultiMouseServer` singleton, polls it every
frame, and re-emits signals:

- `device_connected(device_id: int, info: Dictionary)`
- `device_disconnected(device_id: int)`
- `motion(event: InputEventMouseMotion)`
- `button(event: InputEventMouseButton)`

Each emitted event mirrors the `device_guid` metadata generated by the native
layer so you can keep per-device state:

```gdscript
func _on_motion(event: InputEventMouseMotion) -> void:
    var guid := event.get_meta("device_guid") if event.has_meta("device_guid") else str(event.device)
    _cursors[guid].position += event.relative
```

`attach_to_window()` should be called once per window you want to capture. On
Windows you can pass `0` to use the main window handle; on other platforms we’ll
use the engine default until the libinput backend is complete.
